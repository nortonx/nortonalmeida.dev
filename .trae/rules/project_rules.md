1. The project uses Next.js 15+ with the App Router
2. The project uses Tailwind CSS for styling
3. The project uses shadcn/ui for components
4. The project uses lucide-react for icons
5. The project uses zod for form validation
6. All components are styled using Tailwind CSS
7. The project uses TypeScript for type checking. Never uses plain JavaScript.
8. The project should not use "any" type.
9. The project uses Jest and React Testing Library for testing
10. The project uses Playwright for end-to-end testing
11. The project should follow Next.js, shadcn/ui, and lucide-react guidelines for component usage, as well as best practices and coding standards
12. The project should use TypeScript for all type definitions, including interfaces, types, and enums
13. The project should use TypeScript for all function signatures, including props, state, and event handlers
14. Always use server actions for API requests
15. Always use useActionState for form submissions
16. The project should use the DRY principle as much as possible
17. The project should use the YAGNI principle as much as possible
18. Do not introduce object oriented programming concepts unless necessary
19. The project should use functional components and hooks as much as possible
20. The project should use React Server Components as much as possible
21. When adding client components to server components, always use dynamic imports to avoid blocking the server
